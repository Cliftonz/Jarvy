# PRD-008: Environment Variables Support

## Overview

Add support for defining and managing environment variables in jarvy.toml, enabling automatic shell configuration and `.env` file generation.

## Problem Statement

After installing tools, developers often need to:
1. Set PATH to include new tool locations
2. Configure API keys and credentials
3. Set tool-specific environment variables (GOPATH, NODE_ENV)
4. Update shell rc files (.bashrc, .zshrc)

Currently, Jarvy installs tools but leaves environment configuration to manual steps, blocking ~30% of potential adopters.

## Evidence

- Common post-install manual steps: "Add to PATH", "Set GOPATH"
- Competitors (direnv, dotenv, nix-shell) all handle env vars
- Shell configuration in setup.rs is hardcoded, not user-configurable

## Requirements

### Functional Requirements

1. **Declare env vars in config**: Define variables in jarvy.toml
2. **Generate .env files**: Create project-local .env for tooling
3. **Update shell rc files**: Optionally modify .bashrc/.zshrc
4. **Per-tool env vars**: Some variables tied to specific tools
5. **Secret handling**: Mark sensitive vars, don't log them
6. **Template variables**: Use `$HOME`, `$PWD`, tool paths dynamically

### Non-Functional Requirements

1. Never overwrite existing .env without confirmation
2. Backup shell rc files before modification
3. Support bash, zsh, fish shell syntaxes
4. Cross-platform path handling

## Proposed Config Syntax

```toml
# jarvy.toml

[tools]
node = "20"
go = "1.21"
aws-cli = "latest"

[env]
# Simple key-value
NODE_ENV = "development"
LOG_LEVEL = "debug"

# Template with variable expansion
GOPATH = "$HOME/go"
PATH = "$PATH:$GOPATH/bin:$HOME/.local/bin"

# Per-tool environment (set after tool installs)
[env.go]
GOROOT = "/usr/local/go"
GOPATH = "$HOME/go"

[env.node]
NODE_PATH = "$HOME/.npm-global"

# Secrets (not logged, prompted if missing)
[env.secrets]
AWS_ACCESS_KEY_ID = { prompt = "Enter AWS Access Key" }
AWS_SECRET_ACCESS_KEY = { prompt = "Enter AWS Secret Key", hidden = true }
DATABASE_URL = { from_file = ".secrets/database_url" }

# Shell configuration
[env.config]
shell = "zsh"                    # or "bash", "fish", "auto"
update_rc = true                 # Modify shell rc file
generate_dotenv = true           # Create .env file
dotenv_path = ".env"             # Default location
export_prefix = "jarvy_"         # Comment marker in rc file
```

### Environment Variable Types

```rust
// src/env.rs
#[derive(Deserialize)]
#[serde(untagged)]
pub enum EnvValue {
    // Simple string value
    Simple(String),

    // Complex configuration
    Complex {
        value: Option<String>,
        prompt: Option<String>,
        hidden: Option<bool>,
        from_file: Option<PathBuf>,
        required: Option<bool>,
    },
}

#[derive(Deserialize)]
pub struct EnvConfig {
    #[serde(flatten)]
    pub vars: HashMap<String, EnvValue>,

    #[serde(default)]
    pub config: EnvSettings,

    #[serde(default)]
    pub secrets: HashMap<String, EnvValue>,
}

#[derive(Deserialize)]
pub struct EnvSettings {
    pub shell: Shell,
    pub update_rc: bool,
    pub generate_dotenv: bool,
    pub dotenv_path: PathBuf,
}
```

### Variable Expansion

```rust
// src/env/expand.rs
pub fn expand_variables(value: &str, context: &EnvContext) -> String {
    let mut result = value.to_string();

    // Built-in variables
    result = result.replace("$HOME", &context.home_dir);
    result = result.replace("$PWD", &context.current_dir);
    result = result.replace("$USER", &context.username);

    // Existing environment
    for (key, val) in std::env::vars() {
        result = result.replace(&format!("${}", key), &val);
        result = result.replace(&format!("${{{}}}", key), &val);
    }

    // Tool-provided paths
    for (tool, path) in &context.tool_paths {
        result = result.replace(&format!("$JARVY_{}_PATH", tool.to_uppercase()), path);
    }

    result
}
```

### .env File Generation

```rust
// src/env/dotenv.rs
pub fn generate_dotenv(env_config: &EnvConfig, path: &Path) -> Result<(), EnvError> {
    let mut content = String::new();
    content.push_str("# Generated by Jarvy - do not edit manually\n");
    content.push_str(&format!("# Generated at: {}\n\n", chrono::Local::now()));

    for (key, value) in &env_config.vars {
        let expanded = expand_variables(&value.resolve()?, &context);

        // Quote values with spaces
        if expanded.contains(' ') {
            content.push_str(&format!("{}=\"{}\"\n", key, expanded));
        } else {
            content.push_str(&format!("{}={}\n", key, expanded));
        }
    }

    // Check for existing file
    if path.exists() {
        let existing = std::fs::read_to_string(path)?;
        if !existing.contains("Generated by Jarvy") {
            return Err(EnvError::FileExists(path.to_path_buf()));
        }
    }

    std::fs::write(path, content)?;
    Ok(())
}
```

### Shell RC Update

```rust
// src/env/shell.rs
pub fn update_shell_rc(env_config: &EnvConfig, shell: Shell) -> Result<(), EnvError> {
    let rc_path = match shell {
        Shell::Bash => home_dir().join(".bashrc"),
        Shell::Zsh => home_dir().join(".zshrc"),
        Shell::Fish => home_dir().join(".config/fish/config.fish"),
    };

    // Backup existing
    if rc_path.exists() {
        let backup = rc_path.with_extension("bak.jarvy");
        std::fs::copy(&rc_path, &backup)?;
    }

    let mut content = std::fs::read_to_string(&rc_path).unwrap_or_default();

    // Remove old Jarvy section if exists
    let marker_start = "# jarvy_start";
    let marker_end = "# jarvy_end";
    if let Some(start) = content.find(marker_start) {
        if let Some(end) = content.find(marker_end) {
            content = format!("{}{}", &content[..start], &content[end + marker_end.len()..]);
        }
    }

    // Add new section
    content.push_str("\n# jarvy_start - managed by Jarvy, do not edit\n");

    for (key, value) in &env_config.vars {
        let expanded = expand_variables(&value.resolve()?, &context);
        match shell {
            Shell::Fish => content.push_str(&format!("set -gx {} \"{}\"\n", key, expanded)),
            _ => content.push_str(&format!("export {}=\"{}\"\n", key, expanded)),
        }
    }

    content.push_str("# jarvy_end\n");

    std::fs::write(&rc_path, content)?;
    Ok(())
}
```

### Secret Prompting

```rust
// src/env/secrets.rs
pub fn resolve_secrets(secrets: &HashMap<String, EnvValue>) -> Result<HashMap<String, String>, EnvError> {
    let mut resolved = HashMap::new();

    for (key, config) in secrets {
        let value = match config {
            EnvValue::Complex { prompt: Some(msg), hidden, from_file: None, .. } => {
                if *hidden.as_ref().unwrap_or(&false) {
                    rpassword::prompt_password(msg)?
                } else {
                    dialoguer::Input::new().with_prompt(msg).interact_text()?
                }
            }
            EnvValue::Complex { from_file: Some(path), .. } => {
                std::fs::read_to_string(path)?.trim().to_string()
            }
            _ => continue,
        };

        resolved.insert(key.clone(), value);
    }

    Ok(resolved)
}
```

## CLI Integration

```bash
# Normal setup applies env vars
jarvy setup

# Only configure environment (skip tool installation)
jarvy env

# Generate .env file only
jarvy env --dotenv

# Update shell rc only
jarvy env --shell

# Show what would be set without applying
jarvy env --dry-run

# Export for current shell (use with eval)
jarvy env --export
# Usage: eval $(jarvy env --export)
```

## Risks

1. **Shell rc corruption**: Bad writes could break user shell
   - Mitigation: Always backup, use clear markers, validate syntax
2. **Secret exposure**: Secrets in .env could be committed
   - Mitigation: Add .env to .gitignore automatically, warn user
3. **Path conflicts**: Duplicate PATH entries
   - Mitigation: Deduplicate PATH before writing
4. **Cross-platform paths**: Windows vs Unix paths
   - Mitigation: Use platform-aware path joining
